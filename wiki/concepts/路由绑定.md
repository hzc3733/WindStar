## 定义

路由绑定（Bindings）是多Agent系统中决定消息应该被路由到哪个Agent的规则配置。OpenClaw中通过bindings配置实现消息的智能分发。

## 优先级规则

从高到低依次为：

1. **peer精确匹配**：精确匹配私信或群组ID
   ```json
   { "agentId": "work", "match": { "peer": { "kind": "dm", "id": "5010223608" } } }
   ```

2. **guildId**：Discord服务器级别匹配

3. **teamId**：Slack团队级别匹配

4. **accountId**：渠道账户级别匹配

5. **渠道级匹配**：整个渠道（如telegram）路由到同一Agent

6. **默认Agent**：没有任何匹配时使用默认Agent

## 配置示例

```json
"bindings": [
  { "agentId": "work", "match": { "channel": "telegram" } },
  { "agentId": "family", "match": { "channel": "telegram", "peer": { "kind": "group", "id": "-1001234567890" } } },
  { "agentId": "main", "match": { "channel": "webchat" } }
]
```
